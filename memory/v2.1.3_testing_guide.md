# BulkSMS App v2.1.3 - Permission Fix Testing Guide

## 🎯 What Changed in v2.1.3

**CRITICAL FIX**: The app now **PROPERLY BLOCKS** sending if SMS permission is not granted!

### Previous Bug (v2.1.0 - v2.1.2)
- Permission check would fail silently
- App would continue trying to send SMS even without permission
- Result: `generic_failure` error from Android

### Fixed in v2.1.3
- ✅ Permission check now **stops execution** if permission denied
- ✅ Clear error message: "SMS permission DENIED. Go to Android Settings..."
- ✅ **Permission status banner** appears on dashboard if not granted
- ✅ One-click "Grant Now" button to request permission
- ✅ Better logging to track permission state

---

## 📱 Testing Steps for @Strew2t

### Step 1: Install Fresh APK
1. Download v2.1.3 from bot
2. **Uninstall old version first** (to reset permissions)
3. Install new APK
4. Open the app

### Step 2: Check Permission Status on Launch
When you first open the app and login, you should see:

**If permission NOT granted:**
- 🟠 **Yellow warning banner** appears at top of dashboard:
  ```
  ⚠️ SMS Permission Required - Grant Now
  ```

**If permission IS granted:**
- ✅ No warning banner (all good!)

### Step 3: Grant Permission (if needed)

**Method 1: Via Banner**
- Click "Grant Now" in the yellow banner
- Android will prompt for SMS permission
- Tap "Allow"

**Method 2: Manual**
1. Go to Android Settings > Apps
2. Find "Nomadly SMS"
3. Tap Permissions > SMS
4. Select "Allow"

### Step 4: Test Sending
1. Create a test campaign with **1 contact**
2. Use a valid number with country code: `+1234567890`
3. Set gap time to 10 seconds
4. Tap "Start Sending"

**Expected Outcomes:**

**Scenario A: Permission granted**
- SMS sends successfully ✅
- OR shows specific carrier error (e.g., "no_service", "generic_failure" with helpful hint)

**Scenario B: Permission denied**
- App shows error immediately:
  ```
  SMS permission DENIED. Go to Android Settings > Apps > 
  Nomadly SMS > Permissions and enable SMS.
  ```
- No attempt to send
- No `generic_failure` error

## 📊 Troubleshooting

### Issue: Banner doesn't appear even without permission
- **Cause**: Old APK cached
- **Fix**: Force stop app, clear cache, restart

### Issue: Permission granted but SMS still fails
- **Check**: Can you send manual SMS via native SMS app?
- **If NO**: SIM has no credit or carrier blocked
- **If YES**: Check Railway logs for specific error details

### Issue: "generic_failure" still appears
- **If permission is granted**: This is now a carrier/SIM issue (most likely no SMS credit)
- **Check**: Send manual SMS from device to verify SIM works
- **Detailed error** should now appear in logs with hint
